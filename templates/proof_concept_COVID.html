Introduction

{% blocktranslate trimmed %} Cette preuve de concept utilise le Random Forest pour illustrer le potentiel d’une approche combinant l’intelligence artificielle (IA) et les systèmes d’information géographique (SIG) pour des problématiques de santé publique. Le résultat est loin d’être parfait, car les données ouvertes sur la COVID qui ont alimenté le modèle manquaient de volume et de précision. Des explications seront fournies pour souligner les problématiques et les correctifs à apporter pour un projet de recherche complet. {% endblocktranslate %}

{% blocktranslate trimmed %} Cette technique est très peu utilisée dans la littérature, alors cela demeure un vaste champ à explorer. Cependant, ces résultats préliminaires et l’étude de {% endblocktranslate %} Tiwari et al., {% blocktranslate trimmed %} laisse présager que cette approche pourrait être bénéfique pour la santé publique. {% endblocktranslate %}

{% blocktranslate trimmed %} Les prochaines sections expliquent en détail la technique utilisée pour mieux comprendre les subtilités et les résultats de l’algorithme. Dans un premier temps, les données à prédire, c’est-à-dire les hospitalisations reliées à la COVID par région sociosanitaire, seront analysées. Ensuite, les variables pertinentes à première vue, mais non retenues pour être intégrées à l’algorithme, seront expliquées. Les variables retenues seront justifiées dans la section suivante, puis la performance du Random Forest sera analysée. Un exemple d’un tableau de bord dynamique permettant d’interagir avec le modèle construit suit. Enfin, un parallèle avec mon projet de recherche sur les changements climatiques est présenté. {% endblocktranslate %}

Données à prédire

{% blocktranslate trimmed %} Le {% endblocktranslate %} {% translate "taux d’hospitalisation quotidien relié à la COVID" %} {% blocktranslate trimmed %} du 10 avril 2020 au 14 août 2021 a été utilisé pour les 16 régions sociosanitaires les plus populeuses (le Nunavik et les Terres-Cris-de-la-Baie-James ont été écartés, puisque leur réalité est totalement différente du reste du Québec). Cela représente 7872 observations pour l’algorithme (492 jours x 16 régions). {% endblocktranslate %}

{% blocktranslate trimmed %} Les taux d’hospitalisations variaient entre 0% et 0,064% par région. Afin d’éviter de choisir de façon arbitraire un nombre de classes (par exemple, par quintile), la distribution des données a été étudiée. La méthode du coude (elbow method) a permis de déterminer que 5 classes représentaient bien les données. L’algorithme du Jenks Natural Breaks a permis de séparer les niveaux d’hospitalisation ainsi : {% endblocktranslate %}

    {% blocktranslate trimmed %}
  • Niveau 1 : 0 à 0,0034 % (4 738 région-jours)
  • Niveau 2 : 0,0035 à 0,010 % (2175 région-jours)
  • Niveau 3 : 0,011 à 0,021 % (758 région-jours)
  • Niveau 4 : 0,022 à 0,042% (166 régions-jours)
  • Niveau 5 : 0,043 à 0,064% (35 régions-jour)
    • {% endblocktranslate %}

{% blocktranslate trimmed %} Seule la région de Montréal a atteint un niveau 5 d’hospitalisation (35 jours). Les régions du Saguenay-Lac-Saint-Jean (37 jours), de Montréal (87 jours) et de Laval (42 jours) ont atteint le niveau 4 au cours de la pandémie. Le Bas-Saint-Laurent, l’Abitibi-Témiscamingue, la Côte-Nord, le Nord-du-Québec et la Gaspésie-Iles-de-la-Madeleine n’ont jamais dépassé le niveau 2. {% endblocktranslate %}

{% blocktranslate trimmed %} Il aurait été beaucoup plus intéressant d’avoir des données plus précises (par RLS ou par aires de diffusion) pour avoir une plus grande diversité d’observations, mais elles n’étaient pas disponibles à ma connaissance sous format de données ouvertes. {% endblocktranslate %}

{% blocktranslate trimmed %} Il est important de comprendre les données analysées pour comprendre la performance de l’algorithme développé. Puisqu’une région comme l’Abitibi-Témiscamingue n’a jamais dépassé le niveau 2, il ne faut pas se surprendre si l’algorithme ne prédit jamais un niveau plus élevé pour cette région. Les algorithmes d’apprentissage machine et de réseaux de neurones reconnaissent des schémas qu’ils ont déjà observés.Cette subtilité doit être bien comprise pour pleinement comprendre ce que l’on entend par prédiction dans le tableau de bord. {% endblocktranslate %}

Variables intéressantes non retenues

{% blocktranslate trimmed %} Plusieurs variables en lien avec la pandémie de COVID-19 ont été explorées, mais n’ont pas été retenues pour diverses raisons. {% endblocktranslate %}

{% blocktranslate trimmed %} Dans un premier temps, les {% endblocktranslate %} {% translate "mesures sanitaires répertoriées sur la ligne du temps par l'INSPQ" %} {% blocktranslate trimmed %} ont été intégrées aux premiers essais de modélisation. Cependant, il y avait plusieurs problèmes avec cette approche. Premièrement, certaines mesures comme le port du masque ont été en vigueur pendant presque toute la durée de la pandémie, ce qui occasionnait un manque d’observations sans la mesure pour que l’algorithme puisse tirer des conclusions. {% endblocktranslate %}

{% blocktranslate trimmed %} D’autres mesures comme le couvre-feu n’avaient aucun lien avec les hospitalisations et l’algorithme n’arrivait pas à intégrer ces paramètres. Enfin, des mesures comme la fermeture des commerces étaient trop bien corrélées avec les hospitalisations et écrasaient toutes les autres variables dans le modèle. En effet, cette mesure était mise en place lorsque le gouvernement craignait un débordement des hôpitaux. Ainsi, cette variable a été mise de côté, car un algorithme qui reconnaît une variable étroitement corrélée au phénomène étudié ne sert à rien. Il suffit à un être humain de regarder ladite variable et de tirer ses propres conclusions. {% endblocktranslate %}

{% blocktranslate trimmed %} Le {% endblocktranslate %} {% translate "nombre de cas confirmés" %} {% blocktranslate trimmed %} aurait pu être intéressant, mais puisque les données ouvertes ne permettent pas d’avoir un historique par région, cette avenue n’a pas été explorée. {% endblocktranslate %}

{% blocktranslate trimmed %} Le {% endblocktranslate %} {% translate "taux de variants" %} {% blocktranslate trimmed %} en circulation a aussi été analysé, mais il y avait trop peu d’observations dans le modèle pour que l’algorithme puisse l'utiliser. {% endblocktranslate %}

{% blocktranslate trimmed %} Certaines données auraient aussi pu être intéressantes comme le taux de positivité ou le taux de reproduction (Rt), mais les données historiques par région n’étaient pas disponibles. {% endblocktranslate %}

Variables pertinentes retenues

{% blocktranslate trimmed %} Afin d’évaluer la pertinence des variables, la {% endblocktranslate %} {% translate "valeur SHAP" %} {% blocktranslate trimmed %} a été retenue. Plus la valeur est élevée, plus la variable contribue au modèle. Cela ne veut pas dire forcément un lien de cause à effet, seulement que cette variable est plus ou moins significative dans le résultat final pour ce modèle avec ces variables. Une valeur SHAP n’a pas de signification absolue, elle est relative aux autres valeurs dans le modèle. Ainsi, les variables retenues ont été celles qui étaient les plus significatives et qui impactaient le F1-Score et la précision du modèle. Si une variable pouvait être retirée du modèle sans affecter la performance, elle n’était pas conservée. Un sommaire du résultat est disponible dans la figure ci-bas. {% endblocktranslate %}

{% blocktranslate trimmed %} La variable la plus pertinente dans tous les modèles qui ont été essayés était le taux de contacts sociaux dans le dernier mois basé sur les données de {% endblocktranslate %} {% translate "l'étude CONNECT" %}. {% blocktranslate trimmed %} Il est tout à fait logique de croire qu’une augmentation des contacts sociaux amène à une augmentation des cas infectieux et le modèle suit cette logique. {% endblocktranslate %}

{% blocktranslate trimmed %} Plus d'une centaines de {% endblocktranslate %} {% translate "variables socioéconomiques du recensement de Statistique Canada de 2016" %} {% blocktranslate trimmed %} ont été testées. Les différentes tranches d’âges ou de revenus n’ont pas été un facteur important dans le présent modèle. Cela peut s’expliquer aisément. Le nombre d’observations était trop petit (seulement 16 régions) pour intégrer une grande quantité de variables. Ce phénomène s’appelle le curse of dimensionality. Essentiellement, utiliser plus de variables n’est pas forcément mieux pour un modèle si le nombre d’observations n’est pas conséquent. Par exemple, si un modèle souhaite prédire la couleur d’un objet et que nous avons des dizaines de déclinaisons (bleu pâle, bleu foncé, bleu azur, etc.), le modèle va nécessiter une énorme quantité d’observation pour chaque sous-couleur. Tout rassembler sous la couleur bleue serait alors une meilleure stratégie. {% endblocktranslate %}

{% blocktranslate trimmed %} C’est exactement ce qui a été fait dans ce modèle en prenant la mesure de faibles revenues pour les 65 ans et plus. Cette variable s’est avérée très significative. Il est en effet logique de croire que les personnes âgées sont les plus vulnérables à la maladie et que ceux avec un faible revenu ont plus de chance d’être dans un environnement à risque. {% endblocktranslate %}

La même logique a prévalu pour l’immigration récente, puisque plusieurs travailleurs de premières lignes ou de salariés précaires à risque sont des immigrants.

Enfin, le taux de vaccination a également un impact sur la prédiction, mais moins que ce que l’on pourrait croire. L’une des raisons est que peu d’observations ont été faites avec un haut taux de 2e dose (phénomène récent dans l’historique pandémique). Pour ce qui est de la 1ere dose, elle a bien sûr un impact, mais les premières vagues ont été maîtrisé sans avoir un haut taux de vaccination. Néanmoins, cette variable demeure pertinente pour créer un tableau de bord qui englobe divers scénarios (haut taux de vaccination vs absence de vaccination). Il faut cependant s’attendre à quelques imprécisions dues au faible volume de données. Cette imprécision est quantifiée à la section suivante.

Analyse de la performance du Random Forest

{% blocktranslate trimmed %} Un réseau de neurones aurait pu être utilisé, mais puisque mon serveur n’est pas puissant, je préférais alimenter le tableau de bord avec un Random Forest qui est beaucoup moins demandant en ressource une fois entraînée. Cependant, pour mon projet de doctorat, un réseau de neurones sera utilisé et sera vraisemblablement plus performant. {% endblocktranslate %}

{% blocktranslate trimmed %} Le Random Forest (hyperparamètres : 1024 trees, 3 minimum sample splits, 1 minimum sample leaf) a été entraîné avec 75 % des 7872 observations. Ce dernier a atteint une précision de 91,6 % et un F1-Score de 91,5 % sur les 25 % de données tests. {% endblocktranslate %}

{% blocktranslate trimmed %} Puisque les classes sont fortement déséquilibrées, il ne faut pas se fier sur la précision. En effet, un algorithme qui prédit toujours la classe dominante peut apparaître performant, mais n’être d’aucune utilité pour détecter autre chose que la classe prépondérante. Dans le cas présent, le F1-Score de 91,5 % qui est une moyenne harmonique de la précision et du rappel de chaque classe est intéressant. En effet, si seule la classe dominante avait été prédite adéquatement par l’algorithme, le F1-Score aurait été médiocre. La matrice de confusion ci-bas dénote une performance correcte pour l’ensemble des classes, sans être exceptionnelle. Un manque d’observation explique une performance ordinaire pour les niveaux élevés d’hospitalisation. {% endblocktranslate %}

Parallèle avec mon projet de recherche sur les changements climatiques

{% blocktranslate trimmed %} Le problème avec les données ouvertes de COVID est le faible niveau d’observation pour entraîner l’algorithme (seulement 16 régions et 492 jours). L’accès aux données par RLS ou par aires de diffusion aurait produit des résultats plus intéressants. {% endblocktranslate %}

{% blocktranslate trimmed %} L’avantage d’étudier les hospitalisations en fonction des données de températures estivales réside dans son grand nombre d’observations dans les dernières décennies. De plus, cette problématique est relativement constante dans sa physiopathologie. La pandémie actuelle offre des conditions fort variables d’une vague à l’autre (variants, vaccinations, etc.), ce qui complexifie d’autant plus le travail de l’algorithme. {% endblocktranslate %}

{% blocktranslate trimmed %} Un tableau de bord sur les vagues de chaleur pourrait intégrer des données météorologiques avec des plages de températures variées, un simulateur en fonction de la démographie changeante (comme la proportion de personnes âgées par secteurs) ou bien en fonction du couvert végétal présent. Cette approche comporte plusieurs avantages : {% endblocktranslate %}

    {% blocktranslate trimmed %}
  • Productions d’un grand nombre de cartes en fonction de différents paramètres (vs une carte unique générale de vulnérabilité)
  • Puisque la carte produite par l’IA utilise directement des données d’hospitalisation et de mortalité, le niveau de précision est plus grand. Il n’y a pas de pertes d’information via une analyse en composante principale ou de débats pour choisir des variables et leur pondération.
  • L’intégration d’un grand nombre de variables dont les relations ne sont pas forcément linéaires est possible si les observations sont suffisantes.
  • Possibilité de fournir en temps réel des prédictions pour alimenter un système d’alerte.
  • Une fois une infrastructure mise en place (j’ai codé tout le site web seul avec des logiciels libres, ce n’est ni très coûteux, ni très complexe), l’IA peut constamment s’améliorer en apprenant tous les ans avec de nouvelles données.
  • Une meilleure compréhension des phénomènes peut en découler. Bien qu’on ne puisse pas effectuer de relations de cause à effet, les patrons identifiés par l’IA permettent d’approfondir certains angles qui pourraient être moins évidents de prime à bord.
  • Le même principe peut s’appliquer pour un grand nombre d’aléas (inondations, allergies, feux de forêt) à condition d’adapter les variables pertinentes.
    • {% endblocktranslate %}

{% blocktranslate trimmed %} Il y a bien évidemment des désavantages et l’IA ne peut pas remplacer les techniques traditionnelles d’épidémiologie. Ce n’est d’ailleurs pas le but, l’objectif est d’être complémentaire et d’offrir un outil supplémentaire à la santé publique. Notons, entres autres : {% endblocktranslate %}

    {% blocktranslate trimmed %}
  • L’algorithme ne peut pas extrapoler des données. Par exemple, un algorithme bien entraîné n’aurait pas pu prédire adéquatement le nombre de morts en valeur absolue dans l’importante vague de chaleur que la Colombie-Britannique a vécu cet été. Puisque les changements climatiques apporteront des évènements extrêmes qui n’ont jamais été observés, seul un ordre de grandeur peut être fourni. Ceci étant dit, cela demeure vrai pour n’importe quel outil qui tente d’effectuer des prédictions. L’IA possède l’avantage d’être flexible et d’intégrer rapidement ces nouvelles données.
  • La notion de vie privée est très importante à considérer dans une éventuelle implantation d’une solution basée sur l’IA. Ma méthodologie évalue les données au niveau populationnel et n’utilise pas de données qui permettent d’identifier des individus, ce qui augmente l’adhésion de la population à ce type de solution.
  • À ma connaissance, très peu de recherches sont effectuées pour créer des cartes de vulnérabilités basées sur l’IA et les SIG. La majorité de la littérature scientifique semble s’appuyer sur les analyses en composante principale pour le moment ou ne considérer qu’un seul aspect à la fois. Plus de recherches sont nécessaires pour pleinement évaluer le potentiel de cette approche en contexte de changements climatiques.
    • {% endblocktranslate %}
